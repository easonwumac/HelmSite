Product summary

You should not have to babysit your AI agent from a terminal.

Helm 不是另一個 remote shell，而是把 AI coding session 的狀態整理成你可以快速判斷、 快速介入的控制面板。

Desktop macOS menu bar shell

負責啟動與配置本機 relay。

Relay Rust control plane

處理 state、persistence、auth、WSS。

Mobile iPhone client

即時看 session、approve、steer。

Scope Codex-first

以 structured agent events 為核心。

Why Helm exists

It solves the waiting problem, not just remote access.

不必守在 terminal

session 的狀態、等待點與 approval request 直接送到 iPhone。

看的是 agent intent

不是只看 raw log，而是 turn、檔案變更與即將執行的命令。

支援多 session

同時跑多個倉庫時，仍能用一致的視角追蹤風險與進度。

How it works

A compact flow that keeps the Mac as the source of truth.

01

啟動 relay

macOS menu bar shell 接住本機 Codex 流量，整理成可視化事件。

02

掃碼配對

一次 QR pairing 後，手機即可接收狀態更新與通知。

03

遠端審批與 steer

在手機上批准、拒絕或調整 session，不必回到桌面前。

Current scope

The first version already covers the core supervision loop.

  • 即時 WebSocket relay
  • iPhone 端 live session feed
  • approval workflow 與 turn steering
  • QR code pairing + device token
  • SQLite event history
  • 多裝置與 revocation
  • Local Only / Tailnet 雙模式
  • session forking

Maintenance note
